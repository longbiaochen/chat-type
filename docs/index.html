local-first macOS input layer

按下 F5 说话，文字回到当前光标。

ChatType 是原生 macOS 语音输入工具。它连接这台 Mac 上的 ChatGPT 登录态，录音结束后转写、术语修正、可选润色，再把结果送回你正在输入的地方。不能安全粘贴时，结果保留在剪贴板。

demo

三步完成一段可用输入。

它不是聊天窗口，也不是另一个待切换的编辑器。ChatType 把语音转写放回输入层，让你在 Notes、Mail、Slack、Codex 或浏览器表单里继续写。

1

把光标放好

选择任何可编辑位置。ChatType 会先检查目标是否适合自动粘贴。

2

F5 开始，再按 F5 停止

单一触发键保持稳定：不用记复杂快捷键，也不用切到录音窗口。

3

结果落回输入处

可粘贴时自动回填；目标不安全时保留到剪贴板，手动 Cmd+V 即可。

why it feels different

它理解你是在写东西，而不是只生成一段录音文字。

A

术语先对齐

导入词表后，产品名、文件名、英文大小写和常用纠错会在润色前后各跑一次。

B

长句可选 AI Polish

长段口述可以被整理成更像可执行 prompt 的结构；短句保持快速，不强行润色。

C

粘贴保守

只有检测到可编辑目标才声明成功，避免把文本送到错误窗口。

versus ordinary dictation

普通听写停在转写。ChatType 关心转写之后能不能直接用。

普通听写

  • 结果常常需要复制到另一个工具里二次整理。
  • 专有名词和中英混输容易漂移，需要反复手改。
  • 无法确认当前输入目标是否安全，只能假设粘贴成功。
  • 长段口述里的修正语和重复语会原样留下。

ChatType

  • F5 输入链路就在当前 app，减少切换和复制。
  • 术语词表和确定性修正保护关键字。
  • 检测可编辑目标，失败时改用剪贴板回退。
  • 可选 ChatGPT Auth AI Polish，把长段整理成可执行文本。

model and account path

默认走本机 ChatGPT 登录态，不要求额外 API key。

ChatType 的公开路径是给已经使用 ChatGPT Desktop 的个人用户准备的：通过默认浏览器完成 OAuth，保存 app 自己的本地会话，再调用转写路径。高级 OpenAI-Compatible 路线只作为恢复和自定义场景。

默认

ChatGPT 登录态 + ChatType 本地会话。安装后连接账号，之后用 F5 录音。

可选

AI Polish 只处理适合整理的长段文本，失败不会吞掉原始转写。

高级恢复

OpenAI-Compatible 端点适合自带 key 的用户，不是默认 onboarding。

privacy

权限和数据边界写清楚。

ChatType 是 local-first 桌面工具，但它不是离线模型。录音会通过你连接的 ChatGPT 路径完成转写；本地负责录音、会话、术语、权限检查、粘贴判断和剪贴板回退。

1

麦克风

首次录音时触发系统权限提示。权限修复以 /Applications/ChatType.app 为准。

2

Accessibility

只用于把结果送回当前可编辑目标。没有安全目标时保留到剪贴板。

3

本地配置

术语词表、会话和偏好保存在这台 Mac 的 ChatType 配置目录里。

built to trust

强在效率，也明确说出限制。

ChatType 不是稳定公开 API 的企业级承诺。它是给个人高频输入做的原生工具，依赖本机 ChatGPT 登录态和上游私有转写链路。

A

不要从 dist 运行

真实权限、LaunchServices 和验证路径都绑定安装后的 /Applications/ChatType.app。

B

上游可能变化

如果 ChatGPT Web 会话或私有端点变化，ChatType 会把失败暴露出来，而不是伪装成功。

C

开源可检查

仓库包含源码、测试、打包脚本和发布说明。默认故事保持窄而诚实。
