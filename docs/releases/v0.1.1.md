# ChatType v0.1.1

## What It Does

`ChatType` is a lightweight macOS dictation app for a single fast loop:

1. Press `F5`
2. Speak
3. Press `F5` again
4. Get transcribed text pasted into the focused editor, or copied to the clipboard when paste is not safe

`v0.1.1` defaults to the desktop-login route. If Codex is installed on this Mac and signed in with ChatGPT, `ChatType` can transcribe without asking for any API key.

## Quick Start

1. Install and open Codex on this Mac
2. Sign in with your ChatGPT account
3. Install `ChatType.app` to `/Applications/ChatType.app`
4. Open `/Applications/ChatType.app`
5. Grant microphone permission
6. Grant Accessibility permission if you want auto-paste
7. Focus an editable field and use `F5` to record

## Advanced Recovery

An `OpenAI-Compatible Recovery` route still exists in Settings for troubleshooting and power users. It is not part of the default launch flow.

## Output Path

`v0.1.1` keeps the single-stage STT product path and closes the remaining M1 polish issues.

- no second AI cleanup pass in the default flow
- OpenAI-compatible recovery uses the official transcription `prompt` parameter
- the desktop-login bridge tries the same prompt and falls back automatically if that private path rejects it
- Chinese output is steered to Simplified Chinese and normalized back from Traditional when needed
- optional hidden `transcription.hintTerms` preserve critical terms without another model call

## M1 Fixes

- microphone first-run permission flow now reliably surfaces the system prompt
- Accessibility repair flow is explicit and testable from Settings
- auto-paste works again for Codex-style editors that do not expose a writable AX text node
- occasional Traditional Chinese recognizer drift is normalized back to Simplified Chinese

## Known Limitations

- `ChatType` depends on a local signed-in Codex desktop session in the default path
- the backend path is private and may change upstream
- the release build should be installed to `/Applications/ChatType.app` before first launch or permission repair
- the release build is locally signed and not notarized yet
- the desktop bridge prompt path is best-effort and may need to fall back to plain transcription

## Performance Notes

- auth warm-up now caches the desktop login state for repeated dictations
- per-run latency is written to `~/Library/Application Support/ChatType/latency.jsonl`
- benchmark the packaged path with `./scripts/benchmark_stt.sh <3s.wav> <10s.wav> <30s.wav>`

## Install Notes

Release assets for `v0.1.1` are packaged as:

- `ChatType-0.1.1-macos-arm64.zip`
- `ChatType-0.1.1-macos-arm64.dmg`

Do not run the app directly from the unpacked `dist` copy. Install it to `/Applications/ChatType.app` first so TCC binds permissions to the real runtime path.

If macOS blocks the app:

- right-click `ChatType.app` and choose `Open`
- or run:

```bash
xattr -dr com.apple.quarantine /path/to/ChatType.app
```
