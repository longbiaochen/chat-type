# ChatType v0.1.2

## What It Does

`ChatType` is a lightweight macOS dictation app for one fast loop:

1. Press `F5`
2. Speak
3. Press `F5` again
4. Get transcribed text pasted into the focused editor, or copied to the clipboard when paste is not safe

`v0.1.2` keeps the desktop-login route as the default public path. If Codex is installed on this Mac and signed in with ChatGPT, `ChatType` can transcribe without asking for any API key.

## What Is New In v0.1.2

- manual `Import from TypeWhisper` in Settings
- stronger deterministic terminology alignment after STT
- imported glossary is persisted into ChatType’s own local config
- no second AI cleanup pass added back into the default flow

## Quick Start

1. Install and open Codex on this Mac
2. Sign in with your ChatGPT account
3. Install `ChatType.app` to `/Applications/ChatType.app`
4. Open `/Applications/ChatType.app`
5. Grant microphone permission
6. Grant Accessibility permission if you want auto-paste
7. Optional: import a TypeWhisper terminology snapshot in Settings
8. Focus an editable field and use `F5` to record

## Output Path

`v0.1.2` keeps the single-stage STT product path and strengthens terminology handling without turning the app back into `transcribe -> rewrite`.

- OpenAI-compatible recovery still uses the official transcription `prompt` parameter
- the desktop-login bridge still tries the same prompt and falls back automatically if that private path rejects it
- Chinese output is still steered to Simplified Chinese and normalized back from Traditional when needed
- TypeWhisper imports now provide a stronger deterministic terminology-alignment layer after STT
- hidden `transcription.hintTerms` remain exact-only preservation hints for filenames and other critical literals

## Known Limitations

- `ChatType` still depends on a local signed-in Codex desktop session in the default path
- the backend path is private and may change upstream
- the release build should be installed to `/Applications/ChatType.app` before first launch or permission repair
- the release build is locally signed and not notarized yet
- the desktop bridge prompt path is still best-effort and may need to fall back to plain transcription

## Performance Notes

- auth warm-up still caches the desktop login state for repeated dictations
- per-run latency is written to `~/Library/Application Support/ChatType/latency.jsonl`
- terminology alignment runs locally inside the same dictation pipeline instead of creating a second model stage
- benchmark the packaged path with `./scripts/benchmark_stt.sh <3s.wav> <10s.wav> <30s.wav>`

## Install Notes

Release assets for `v0.1.2` are packaged as:

- `ChatType-0.1.2-macos-arm64.zip`
- `ChatType-0.1.2-macos-arm64.dmg`

Do not run the app directly from the unpacked `dist` copy. Install it to `/Applications/ChatType.app` first so TCC binds permissions to the real runtime path.

If macOS blocks the app:

- right-click `ChatType.app` and choose `Open`
- or run:

```bash
xattr -dr com.apple.quarantine /path/to/ChatType.app
```
