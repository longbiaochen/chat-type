## Current Operating Logic

- Runtime priority: official Chrome plugin -> Computer Use -> legacy diagnostic scripts.
- If Computer Use returns `Transport closed`, reset stale SkyComputerUse processes once and retry; if it still fails, record `tool_unavailable` and stop live operations.
- Morning accepted result: metrics collected, comments reviewed, or blocked safely with exact category.
- Evening accepted result: identifiable draft prepared, publish/review state verified, or blocked safely with exact category.
- Never count missing data as 0. Only write 0 when a visible platform row explicitly shows 0.
- If the visible creator account, note count, or note titles do not match known ChatType history, classify the run as `account_view_mismatch` and stop ChatType metric collection.
- Draft count alone is not evidence. A draft must show ChatType title/body/asset/preview.
- Chrome plugin is enough for text/page-state checks; Computer Use is required for file picker, image upload, visual proof, draft-save proof, publish/review proof, and UI disagreement.
- Stop and report on login, QR/SMS/verification, real-name, phone binding, security warning, paid promotion, platform penalty, public reply, delete, or final publish confirmation unless fresh authorization for that action exists in the active run.
